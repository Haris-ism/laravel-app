# 課題4｜Blade組み込みの基礎（フォーム・検証・リダイレクト）

**区分**: 基礎編 ／ **AI利用**: 🚫 **禁止・自力** ／ **想定時間**: 2日

## 目的
**サーバー側でHTMLを生成して返す「組み込み」**の作り方を、素のBladeだけで一通り身につける。
（Reactの「APIから取得して描画」とは逆の、Laravelが得意とする流れ。ここが今回いちばん新しく学ぶ部分）

> この課題では **Livewireを使いません。** まず素のBladeで作れることが土台になります。

## 🔑 この課題で学ぶLaravel用語
（調べる時のキーワード。「使い方を身につける」対象です）
- **Bladeテンプレート（`.blade.php`）** — HTML＋`{{ }}`＋`@if`/`@foreach` で画面を作る
- **レイアウト / コンポーネント（`<x-layout>`, `<x-...>`, スロット）** — 画面の共通化・部品化
- **`view()` を返すコントローラー** — JSONではなくHTML画面を返す
- **フォーム（`@csrf` / `@method`）** — 送信とCSRF対策、PUT/DELETEの指定
- **サーバー側バリデーション表示（`$errors` / `@error` / `old()`）** — エラー表示と入力保持
- **リダイレクトとフラッシュメッセージ（`redirect()->route()->with()` / `session('status')`）**

## やること
現状のブログを、**JS＋APIではなく、素のBladeのサーバー描画**で作り直す。
1. 一覧・詳細・作成フォーム・編集フォームの画面をBladeで作る（`@foreach` で一覧、`<x-layout>` で共通化）。
2. コントローラーは `view(...)` と `redirect(...)` を返す形にする（JSONを返さない）。
3. フォーム送信 → FormRequestで検証 → **失敗時は入力を保持して同じ画面にエラー表示**（`old()` / `@error`）。
4. 成功時は一覧へ**リダイレクトし、フラッシュメッセージ**（「保存しました」）を出す。
5. `@vite` でCSS/JSを読み込む（既存の設定を流用）。

## 提出物
- 一覧・詳細・作成・編集の各Bladeビュー
- `view()` / `redirect()` を返すコントローラー
- 検証エラー時に `old()` と `@error` が効いている画面

## 受け入れ基準
- [ ] JSでAPIを叩かず、ページ遷移とフォーム送信だけで一通り操作できる
- [ ] 入力エラー時に、入力値が消えずエラーが表示される
- [ ] 保存後にリダイレクトされ「保存しました」等が出る
- [ ] レイアウト/コンポーネントで共通部分がまとまっている
