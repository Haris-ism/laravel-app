# 課題7｜Livewire + Alpine（動的UI）

**区分**: 応用編 ／ **AI利用**: 🚫 **禁止・自力** ／ **想定時間**: 2日

## 目的
**JSをほぼ書かずに動的UIを作る Livewire** を理解する。あわせて **「素のBladeで足りる所」と「Livewireを使う所」の使い分け**を身につける。

> Livewireは**素のBladeの上**に成り立ちます。課題4の素Bladeが土台。**静的・単純な画面は素のBladeのまま**にし、動的な所だけLivewireにするのが正解です。

## 🔑 この課題で学ぶLaravel用語
（調べる時のキーワード。「使い方を身につける」対象です）
- **Livewire** — PHP＋Bladeで動的UIを作るLaravel純正ライブラリ。裏で自動的にサーバーと通信し、変わった所だけHTMLを差し替える
- **Livewireコンポーネント（クラス＋Bladeビュー）／ `wire:model` / `wire:click`**
- **Alpine.js（`x-data` / `@click` / `x-show`）** — ブラウザ側の小さな動き（開閉・トグル）
- **状態の置き場所の違い** — React＝ブラウザ側 / Livewire＝**サーバー側（PHP）**

## やること
1. ブログ一覧に **Livewireでライブ検索**（入力するたびに、ページ再読み込みなしで結果が絞り込まれる）を実装。
2. **インライン編集**など、もう1つ動的な操作をLivewireで作る。
3. モーダルの開閉など**小さなUI状態はAlpine**で書く（Livewireを使わない判断も示す）。
4. 「どこを素のBlade／どこをLivewire／どこをAlpineにしたか」と**その理由**をメモ。
